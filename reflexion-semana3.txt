Es importante que los proyectos de software mantengan un registro de cambios usando herramientas como Git porque esto ayuda a organizar mejor el trabajo y a tener control sobre todo lo que se va modificando en el proyecto. Cada vez que se realiza un cambio en el código o en algún archivo, este puede quedar guardado para consultar después qué se hizo y en qué momento se realizó. Esto es útil porque si ocurre un error o algo deja de funcionar correctamente, se puede volver a una versión anterior sin perder todo el avance. También facilita el trabajo entre varias personas, ya que cada integrante puede aportar cambios sin afectar el trabajo de los demás. Además, llevar un historial permite entender cómo ha evolucionado el proyecto y mantener un proceso más ordenado, evitando confusiones y ayudando a que el desarrollo sea más seguro y eficiente incluso cuando se manejan conocimientos básicos de programación.
